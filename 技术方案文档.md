# AI校园建筑识别项目技术实施方案

## 1. 项目概述

### 1.1 项目目标
开发一个"AI校园建筑识别"系统，通过设备摄像头实时识别校园内的物体（包括垃圾和校园建筑），准确识别物体类别并提供对应的垃圾分类建议。

### 1.2 核心功能
- 实时摄像头画面捕获与处理
- 校园建筑和垃圾的实时识别
- 垃圾分类建议生成
- 友好的用户界面

### 1.3 技术挑战
- 现有多模态模型缺乏对本校特定建筑的识别能力
- 实时识别的性能要求
- 数据集构建与标注

## 2. 技术栈选择

### 2.1 模型选择
- **基础模型**：YOLOv8 (You Only Look Once)
  - 优势：实时性能好，精度高，支持物体检测和分类
  - 适用场景：实时摄像头识别，需要快速响应

### 2.2 开发环境
- **前端**：HTML5 + JavaScript + Bootstrap
- **后端**：Python 3.9+
- **AI框架**：PyTorch
- **依赖库**：
  - OpenCV (图像处理)
  - Ultralytics YOLO (物体检测)
  - Flask (Web服务)
  - TensorFlow Lite (移动端部署可选)

## 3. 数据集构建与预处理

### 3.1 数据收集
- **校园建筑照片**：已准备的校园建筑照片
- **垃圾图像**：收集常见垃圾类型的图像
- **扩充数据**：使用数据增强技术扩充数据集

### 3.2 数据标注
- **标注工具**：LabelImg
- **标注格式**：YOLO格式（txt文件）
- **类别定义**：
  ```
  0: 教学楼
  1: 图书馆
  2: 体育馆
  3: 行政楼
  4: 宿舍楼
  5: 可回收垃圾
  6: 厨余垃圾
  7: 有害垃圾
  8: 其他垃圾
  ```

### 3.3 数据预处理
- 图像大小统一调整为640x640
- 数据增强：旋转、缩放、翻转、亮度调整
- 数据分割：训练集80%，验证集10%，测试集10%

## 4. 模型训练与微调

### 4.1 迁移学习策略
- 使用YOLOv8预训练模型
- 在自定义数据集上进行微调
- 冻结底层权重，只训练顶层分类器

### 4.2 训练参数
- 学习率：0.01
- 批量大小：16
- 训练轮次：100
- 优化器：Adam

### 4.3 模型评估
- 评估指标：mAP (mean Average Precision)
- 目标：mAP@0.5 > 0.85

## 5. 实时摄像头调用与图像处理

### 5.1 摄像头捕获
- 使用OpenCV的VideoCapture API
- 帧率控制：30fps
- 画面分辨率：1280x720

### 5.2 图像处理流程
1. 读取摄像头帧
2. 图像预处理（ resize, 归一化 ）
3. 模型推理
4. 结果后处理（ 非极大值抑制 ）
5. 绘制识别结果

### 5.3 性能优化
- 使用GPU加速（如果可用）
- 模型量化（INT8）
- 批处理优化

## 6. 物体识别与分类建议生成

### 6.1 识别逻辑
- 对每个检测到的物体：
  1. 确定物体类别
  2. 计算置信度
  3. 生成分类建议

### 6.2 垃圾分类建议
- 可回收垃圾：建议投放到蓝色垃圾桶
- 厨余垃圾：建议投放到绿色垃圾桶
- 有害垃圾：建议投放到红色垃圾桶
- 其他垃圾：建议投放到灰色垃圾桶

## 7. 用户界面设计

### 7.1 界面布局
- 顶部：标题栏
- 中间：摄像头实时画面
- 底部：识别结果和分类建议

### 7.2 交互设计
- 开始/停止按钮
- 切换前后摄像头
- 识别历史记录
- 垃圾分类指南

## 8. 技术难点及解决思路

### 8.1 难点1：校园建筑识别
- **问题**：现有模型对特定校园建筑识别准确率低
- **解决思路**：
  - 收集更多校园建筑样本
  - 进行数据增强
  - 微调预训练模型

### 8.2 难点2：实时性能
- **问题**：移动设备上实时识别性能受限
- **解决思路**：
  - 使用轻量级模型（YOLOv8n）
  - 模型量化
  - 硬件加速

### 8.3 难点3：环境光照变化
- **问题**：不同光照条件下识别准确率波动
- **解决思路**：
  - 数据增强（模拟不同光照）
  - 图像预处理（自适应亮度调整）
  - 鲁棒性训练
